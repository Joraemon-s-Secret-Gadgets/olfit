# 에러 핸들링 및 폴백 전략 (Error Handling & Fallback Strategy)

Olfít 백엔드는 다수의 외부 AI API(NVIDIA, OpenAI, Pinecone)에 의존하는 복잡한 파이프라인을 가지고 있습니다. 외부 시스템 장애 시에도 서비스가 중단되지 않도록 단계별 **강력한 폴백(Robust Fallback)** 메니지즘이 설계되어 있습니다.

## 1. VLM 분석 장애 대응 (VLEngine Fallback)

이미지 분석 서비스(NVIDIA NIM)의 타임아웃, 할당량 초과 또는 파싱 오류 발생 시의 대응입니다.

- **장애 유형**: API 호출 실패, 비정상 JSON 응답, 네트워크 지연.
- **폴백 로직**: `DUMMY_RESULT` 반환.
  - 미리 정의된 "기본 스타일"과 "범용적인 무드"를 담은 분석 객체를 즉시 반환하여 이후 파이프라인(아우라 계산 등)이 중단 없이 작동하게 합니다.
- **사용자 경험**: 분석 속도가 매우 빠르며, 최소한 사용자가 선택한 '노트(취향)'를 기반으로 한 추천은 정확하게 수행됩니다.

---

## 2. 벡터 검색 장애 대응 (Pinecone/OpenAI Fallback)

벡터 임베딩 생성(OpenAI) 또는 벡터 검색(Pinecone) 장애 시의 대응입니다.

- **장애 유형**: API Key 만료, 서비스 다운, 임베딩 모델 호출 실패.
- **폴백 로직**: **MySQL 기반 5축 아우라 필터링**.
  1. 사용자의 5축 중 가장 점수가 높은 **메인 계열(Main Family)**을 추출합니다.
  2. MySQL DB에서 해당 계열에 속하는 향수 후보군(50~100개)을 1차 필터링합니다.
  3. 필터링된 후보군을 대상으로 DB에 저장된 아우라 점수만을 이용해 코사인 유사도를 계산하여 순위를 산출합니다.
- **사용자 경험**: 시맨틱 검색(감성 매칭)의 정교함은 다소 낮아질 수 있으나, 수치적으로 가장 잘 어울리는 제품을 추천하는 핵심 가치는 유지됩니다.

---

## 3. 시스템 모니터링 및 로깅

- **Exception Logging**: 모든 외부 API 호출은 `try-except` 블록으로 감싸져 있으며, 에러 발생 시 상세 스택 트레이스를 서버 로그에 기록합니다.
- **Response Validation**: API 응답이 돌아오더라도 필수 필드가 누락된 경우 즉시 폴백 로직을 가동하여 시스템 전체의 정적 타입 안정성을 보장합니다.

---

## 4. 요약 가이드

| 장애 지점 | 감지 방법 | 폴백 동작 | 영향도 |
| :--- | :--- | :--- | :--- |
| **NVIDIA VLM** | API Timeout / Error | 더미 데이터(`DUMMY_RESULT`) 주입 | 이미지 분석 내용이 일반화됨 |
| **OpenAI Embedding** | API Exception | DB 기반 필터링 모드 전환 | 시맨틱 매칭 품질 저하 |
| **Pinecone Search** | Client Exception | DB 기반 필터링 모드 전환 | 검색 속도 소폭 상승, 다양성 감소 |
