# 향수 추천 로직 상세 (Recommendation Logic v4.9.0)

Olfít의 핵심 엔진은 사용자의 시각적 감성과 기술적 취향을 결합하여 최적의 향기를 도출하는 **실시간 하이브리드 추천 파이프라인**을 기반으로 동작합니다.

## 1. 4단계 통합 추천 파이프라인

단일 알고리즘의 한계를 극복하기 위해, 시스템은 시각 인지부터 의미론적 검색, 수치적 재정렬에 이르는 4단계 공정을 거칩니다.

### 1.1 시각 감성 분석 (Vision Stage)
- **도구**: NVIDIA NIM (Gemma-VLM)
- **역할**: 사용자가 업로드한 OOTD 이미지에서 색상, 사물, 스타일, 무드를 구조화된 데이터(JSON)로 추출합니다.
- **핵심**: 단순 묘사가 아닌, 향수 도메인 매핑이 가능한 '트리거 키워드'를 우선적으로 확보합니다.

### 1.2 아우라 스코어링 및 융합 (Aura Fusion Stage)
- **도구**: `AuraService`, `mapper.py`
- **역할**: 추출된 시각 키워드(60%)와 사용자가 직접 선택한 선호 노트(40%)를 가중 결합하여 **5축 아우라 벡터**를 생성합니다.
- **수학적 모델**: 모든 벡터는 코사인 유사도 연산을 위해 **L2 정규화**를 거치며, UI 표시를 위해 **Ref Max 스케일링**이 적용됩니다.

### 1.3 시맨틱 검색 및 추출 (Retrieval Stage)
- **도구**: OpenAI Embedding (`text-embedding-3-small`), Pinecone Vector DB
- **역할**: 사용자의 아우라 프로필을 바탕으로 생성된 **대칭형 RAG 쿼리**를 사용하여 Pinecone에서 유사도가 높은 상위 30개 후보군을 추출합니다.
- **전략**: 인덱싱된 문서와 검색 쿼리의 문장 구조를 동일하게 맞추는 **Symmetric RAG** 전략을 통해 매칭 신뢰도를 극대화합니다.

### 1.4 하이브리드 재정렬 (Hybrid Re-ranking Stage)
- **도구**: `RecommendationService`
- **역할**: 추출된 후보군을 대상으로 **[아우라 유사도 50% + RAG 점수 50% + 노트 가산점]** 공식을 적용하여 최종 추천 순위(Top 5)를 결정합니다.
- **개인화**: 사용자의 패션 스타일과 제품의 특성을 연결하는 **스토리텔링형 추천 사유(Match Reason)**를 동적으로 생성합니다.

## 2. 시스템의 기술적 차별점

### 2.1 감성과 수치의 조화 (Dual-Engine)
단순히 "이 옷에는 이 향수"라는 식의 단편적 매칭이 아니라, 사용자의 스타일이 가진 **추상적 무드(RAG)**와 향수가 가진 **객관적 성분 데이터(Aura)**를 동시에 고려하여 추천의 설득력을 높였습니다.

### 2.2 강력한 가용성 (Robust Fallback)
외부 AI API 장애 시에도 서비스가 중단되지 않도록, MySQL 기반의 아우라 계열 필터링 경로를 상시 대기시켜 시스템의 안정성을 확보했습니다.

### 2.3 UX 최적화 매핑 (User-Centric Mapping)
내부 시스템의 기술 명칭(Oriental, Floral)을 사용자에게 친숙한 명칭(Amber/앰버, Floral/플로랄)으로 자동 치환하여 전달함으로써 리포트의 가독성을 극대화했습니다.
